# 心情怪兽收容所视觉设计规范 V0.1

> 本文档是项目 UI、视觉资产、页面排版和响应式适配的执行基线。新增页面、修改组件或根据截图微调前必须先阅读。实际代码与本文冲突时，应在同一次修改中同步更新本文档，避免出现“代码已变、规范仍旧”的情况。

## 1. 设计目标

视觉关键词：**软萌、治愈、通透、有一点魔法感，但不幼稚**。

页面应让用户感到：

- 情绪被温柔接住，而不是被诊断或教育。
- 怪兽可爱但不喧宾夺主，核心行动始终清晰。
- 信息量较多时仍然轻盈，不卡、挤、乱。
- 在不同手机上保持同一视觉关系，而不是简单等比缩放。

视觉气质建议保持：**温柔 60% + 灵动 25% + 可爱 15%**。

## 2. 不可破坏的视觉约束

以下规则为 P0，未经明确确认不得改变：

1. 首页品牌标题固定使用 `src/assets/brand/home-wordmark.png`。
2. 不使用系统字体、普通粗体或 CSS 逐字变形重新绘制首页品牌标题。
3. 品牌标题、副标题、紫色收容员和气泡必须组成一个完整的首页 Hero 区域。
4. 气泡文字不得重叠、截断或被下一张卡片遮挡；气泡与后续卡片至少保留 20px 的视觉安全距离。
5. 有底部菜单的页面必须使用 `TabPageLayout`；只允许内容区滚动，菜单始终锁定在屏幕底部。
6. 主操作按钮保持紫色梦幻玻璃风格，不改成普通纯色按钮或系统按钮。
7. 页面优先适配微信小程序常见手机尺寸，不能只根据单张截图写死位置。

## 3. 设计基础

### 3.1 颜色

全局基础变量位于 `src/app.less`。

| 角色 | 变量 / 色值 | 用途 |
|---|---|---|
| 奶油背景 | `--cream: #fffaf2` | 页面底色、暖白按钮文字 |
| 主文字 | `--ink: #2b1648` | 标题、重要正文 |
| 次文字 | `--ink-soft: #665873` | 说明、辅助信息 |
| 主薰衣草紫 | `--lavender: #b197fc` | 品牌氛围、装饰、选中态 |
| 深紫 | `--lavender-deep: #7049cf` | 强调、主要交互 |
| 薄荷绿 | `--mint: #a5efd2` | 正向状态、快捷标签 |
| 天空蓝 | `--sky: #91cdff` | 渐变、轻状态 |
| 桃粉 | `--peach: #ffb8c3` | 情绪标签、柔和提醒 |
| 奶油黄 | `--yellow: #ffd881` | 星星、暖色点缀 |
| 品牌副标题紫 | `#9a75b9` | 首页品牌副标题 |

使用规则：

- 单个页面以紫色为视觉主轴，绿、蓝、粉、黄只作小面积辅助。
- 正文优先使用深紫黑，不使用纯黑 `#000`。
- 大面积背景保持低饱和、低对比；高饱和颜色留给角色、主按钮和状态提示。
- 同一卡片内彩色强调不超过 3 种，避免糖果色过载。

### 3.2 字体与字号

正文默认字体：

```css
-apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif
```

首页品牌标题是视觉资产，不属于普通文字系统：

- 固定素材：`src/assets/brand/home-wordmark.png`
- 原始尺寸：760 × 565，透明 PNG。
- 常规显示宽度：390px。
- 窄屏显示宽度：360px。
- 紧凑屏显示宽度：350px。
- 不叠加 CSS 字体、描边、旋转或重新排字。

正文层级使用以下范围。项目源代码使用 Taro 设计稿单位，由构建工具转换，不手动重复换算 rpx：

| 层级 | 建议字号 | 字重 | 示例 |
|---|---:|---:|---|
| 页面/大卡片标题 | 28–34px | 800–900 | 今日热门怪兽、输入卡标题 |
| 正文/气泡 | 24–28px | 500–650 | 收容员气泡、说明正文 |
| 按钮文字 | 30–34px | 800–900 | 开始收容 |
| 标签/辅助信息 | 19–22px | 500–700 | ONGOING、隐私提示、数据量 |
| 底栏文字 | 25px | 650；激活态 800 | 首页、记录、发现、我的 |

约束：

- 不用单纯加粗来制造“可爱”，可爱感主要来自品牌素材、圆角、色彩、留白和角色。
- 普通正文最多使用 3 档字重，避免页面内到处都是粗体。
- 小字号不能用低对比浅紫；辅助文字仍需清晰可读。

### 3.3 圆角

| 组件 | 圆角 |
|---|---:|
| 大玻璃卡片 | 36px |
| 输入框 | 25px |
| 小怪兽卡 | 26px |
| 气泡 | `28px 28px 8px` |
| 状态小卡 | 20px |
| 胶囊按钮/标签 | 999px |
| 圆形入口 | 50% |

圆角越大代表组件越轻、越亲和。信息密集的小组件不能无差别使用 36px 大圆角。

### 3.4 阴影与玻璃感

玻璃卡片基础样式以 `.glass-card` 为准：

- 半透明暖白渐变背景。
- 2px 淡紫描边。
- 柔和外阴影 + 很轻的内部紫色光晕。
- 不使用纯灰重阴影，不制造悬浮面板的商务感。

同一组件最多使用：

- 1 组主要外阴影。
- 1 组内部高光或光晕。
- 只有主按钮和中央收容入口允许增加第二层外发光。

## 4. 间距与页面骨架

### 4.1 页面水平安全区

- 常规页面左右边距：36px。
- 屏幕宽度 `≤ 360px`：左右边距降为 26px。
- 顶部必须结合状态栏和微信胶囊按钮动态计算，不能写死一个固定安全区高度。
- 页面内容底部必须为固定导航预留空间，不能被导航遮挡。

### 4.2 垂直节奏

推荐间距等级：

- 8–12px：图标与文字、标签内部、紧密信息关系。
- 14–20px：同一卡片内模块。
- 20–28px：卡片与卡片、标题与正文。
- 32–48px：页面主要区块。

规则：

- 元素视觉上重叠时，先修正父容器高度和正常文档流，不用持续增加 `z-index` 掩盖问题。
- 负边距只用于有明确视觉目的的轻微衔接，默认不得超过 8px。
- 绝对定位组件必须为后续内容留出真实占位或安全距离。

## 5. 首页 Hero 规范

首页顶部由四部分组成：微信顶部安全区、品牌标题、副标题、紫色收容员与气泡。

### 5.1 顶部操作区

- 高度由 `statusBarHeight + navBarHeight` 动态组成。
- “我的怪兽”入口位于微信胶囊左侧安全区域，不与胶囊抢占空间。
- 入口使用暖白玻璃胶囊，角色头像 + 文字 + 箭头保持单行。

### 5.2 品牌标题与副标题

- 标题只使用 `BrandLockup` 组件和固定品牌 PNG。
- 副标题固定为“接纳每一种心情 · 陪伴每一个你”。
- 副标题居中、单行、浅紫色，两端使用小圆点辅助平衡。
- 常规字号 22px；窄屏 20px；紧凑屏 19px。
- 标题与副标题属于同一组件，不在页面中分别定位。

### 5.3 紫色收容员

- 角色素材固定为 `src/assets/monsters/shelter-guide.png`。
- 常规容器 390px，角色 370px。
- 紧凑屏容器 350px，角色 330px。
- 角色可使用轻微上下漂浮动画：周期约 4.3 秒，位移不超过 10px。
- 动画不得影响容器占位，也不能造成气泡或卡片跟随抖动。

### 5.4 收容员气泡

- 文案使用两个独立的 `Text` 行，不在一个文本节点中使用换行符强制排版。
- 当前结构：
  - 第一行：“我是紫色收容员”
  - 第二行：“陪你把下一步做小～”
- 常规字号 24px；紧凑屏 22px；行高 1.42。
- 气泡使用暖白半透明背景、淡紫边框和柔和阴影。
- 尾巴和心形装饰属于气泡整体，必须完整可见。
- 气泡底部到 `active-agent-card` 或 `capture-card` 顶部的实际视觉距离不得小于 20px。

### 5.5 Hero 高度

- 常规最小高度：510px。
- 紧凑屏最小高度：450px。
- 不允许为了“塞进一屏”无限压缩标题和气泡；优先让内容区正常滚动。

## 6. 组件规范

### 6.1 主操作按钮

首页“开始收容”按钮是品牌级 CTA，固定特征：

- 薰衣草紫到柔紫的横向渐变，中心略深。
- 乳白色玻璃外圈与内部高光边。
- 少量白色、粉白色星光，不使用密集粒子。
- 文字为暖白色，带轻紫描边/投影，必须清晰可读。
- 右侧使用暖白圆形箭头，箭头为紫色。
- 常规最小高度 96px；紧凑屏 82px。
- 点击态只做轻微缩放和透明度变化，不做强闪烁。

普通页面主按钮可复用 `.primary-button`，但首页 CTA 以 `.capture-button` 为最终样式来源。

### 6.2 玻璃卡片

- 使用 `.glass-card`，不要在每个页面重新发明一套白色卡片。
- 卡片之间建议保持 18–24px 间距。
- 卡片边框和阴影保持轻，不使用纯白实体块叠加深灰阴影。

### 6.3 快捷情绪标签

- 使用胶囊形，单行展示，可水平滚动。
- 蓝、粉、绿、黄仅作为低饱和背景区分，不代表医疗或严重程度。
- 文字保持深色，不能因背景变浅而降低可读性。

### 6.4 底部导航

- 固定五栏：首页 / 记录 / 中央收容 / 发现 / 我的。
- 有底栏的页面统一使用 `TabPageLayout`。
- `.tab-page-shell` 固定视口高度并隐藏外层滚动。
- `.tab-page-scroll` 是唯一纵向内容滚动容器。
- `.bottom-nav` 固定在底部并处理 `safe-area-inset-bottom`。
- 中央收容按钮以 94px 紫色玻璃圆球呈现，完整收在底栏内部；白色加号使用 CSS 绘制，保证边缘清晰，不使用带透明留白的位图。
- 普通导航图标统一为 54px 视觉框，未激活图标使用约 3px 的灰紫线条；不得单独放大某个普通图标。
- 底栏基础高度为 106px，另叠加设备底部安全区；顶部使用 30px 圆角，背景使用暖白偏紫的半透明渐变和轻顶部阴影。
- 激活态使用深紫文字和对应激活图标；未激活态使用灰紫。

禁止：

- 把底栏放入页面滚动内容中。
- 每个页面复制一套导航代码。
- 使用系统 TabBar 与当前自定义导航混用。

## 7. 响应式规则

主页由 `getHomeViewport()` 根据设备信息添加响应式类：

- `home-page--narrow`：屏幕宽度 `≤ 360px`。
- `home-page--compact`：屏幕高度 `≤ 760px`，或高宽比 `≤ 1.9`。

响应式调整优先级：

1. 保证微信胶囊、标题、角色和气泡不互相遮挡。
2. 保证输入区域和按钮可完整操作。
3. 保证底部导航固定且不遮住内容。
4. 最后才缩小字号和角色尺寸。

至少检查以下视口：

| 类型 | 建议尺寸 |
|---|---|
| 小屏 | 320 × 568 |
| 窄屏主流 | 360 × 800 |
| iPhone 标准 | 375 × 812 |
| 当前主要基准 | 390 × 844 |
| 大屏 | 430 × 932 |

首页必须分别验证“有进行中任务卡”和“无进行中任务卡”两种状态。

## 8. 图标、角色和装饰

- 导航图标使用 `src/assets/icons/nav/` 中的项目原创资产。
- 怪兽运行时素材使用 `src/assets/monsters/`，高清原稿使用 `artwork/monsters/`。
- 不混用写实照片、扁平商务图标和当前 3D 软萌怪兽风格。
- 装饰星星、爱心、圆点的作用是平衡留白，不承载核心信息。
- 单屏装饰密度要低；装饰不得压住按钮、正文或微信系统区域。

## 9. 动效与反馈

- 页面动效以 160–220ms 的点击反馈为主。
- 角色呼吸/漂浮可使用 4–5 秒慢循环。
- 用户点击中央收容入口时可使用轻触觉反馈。
- 避免持续闪烁、快速弹跳、大面积模糊移动。
- 动效必须尊重内容布局，不能改变固定底栏位置。

## 10. 文案与视觉关系

- 气泡文案保持短句，一行表达一个意思。
- 卡片标题优先控制在一行；必要时最多两行，不使用超小字号硬塞。
- 主 CTA 使用动作词，如“开始收容”“继续行动”，避免抽象口号。
- 安全回应页面停止娱乐化怪兽语气，视觉色彩也应降低活泼程度。

## 11. 开发与验收清单

每次 UI 修改至少完成：

- [ ] 未替换或重新绘制首页品牌标题素材。
- [ ] 微信顶部胶囊区域无冲突。
- [ ] 标题、副标题、角色、气泡关系与本规范一致。
- [ ] 气泡文字无重叠、截断、遮挡。
- [ ] 气泡与下一张卡片间距不少于 20px。
- [ ] 主按钮文字清晰，玻璃边框和星光不过度。
- [ ] 底部导航在滚动时保持固定。
- [ ] 内容末尾不被底部导航遮挡。
- [ ] 窄屏、紧凑屏和标准屏至少各验证一次。
- [ ] 有任务卡与无任务卡两种首页状态均验证。
- [ ] `npm run typecheck` 通过。
- [ ] `npm run build:weapp` 通过。

视觉调整不能只以“编译通过”为验收结果，必须同时检查实际设备或等效手机视口的页面关系。

## 12. 变更流程

当设计需求与本规范不一致时：

1. 先明确是局部微调还是设计方向变更。
2. 局部微调直接修改组件，并同步更新相关数值和验收项。
3. 品牌标题、主按钮、底部导航或整体风格发生变化时，必须升级规范版本。
4. 修改后保留一张标准视口截图作为对照，并完成微信小程序构建验证。

当前视觉实现入口：

- 全局基础：`src/app.less`
- 通用组件：`src/components/components.less`
- 品牌标题：`src/components/BrandLockup.tsx`
- 首页布局：`src/pages/index/index.tsx`、`src/pages/index/index.less`
- 固定底栏：`src/components/TabPageLayout.tsx`、`src/components/BottomNav.tsx`
