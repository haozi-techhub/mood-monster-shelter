## 1. 设计理念

### 1.1 风格关键词
**怪诞可爱(Grotesque-cute)** + **成人手账(Adult Scrapbook)** + **收容所档案感(Shelter Archive)** — 不要太儿童化,也不要太心理咨询化。

视觉风格是 **Tactile Glassmorphism(触感玻璃拟态)**:软糯果冻质感 + 物理手账结构(和纸胶带、印章、细墨线描边),营造「安全港」氛围。

> 产品定位金句:「成人版宝可梦图鉴 + 情绪手账 + 怪兽收容档案」。

### 1.2 语气 → 视觉映射
产品语气「温柔 60% + 毒舌 25% + 可爱 15%」(来源:`prd` §14、`CLAUDE.md` §6),映射到视觉:

| 语气 | 占比 | 视觉体现 |
|---|---|---|
| 温柔 | 60% | 奶油暖白底、柔和果冻渐变、圆角 squircle、低对比投影、留白 |
| 毒舌 | 25% | 高对比印章/贴纸(`CAPTURED`/`TOXIC`)、Space Grotesk 机械感编号、5–10° 倾斜打破规整 |
| 可爱 | 15% | 吉祥物、爱心/星星贴纸、果冻 Q 弹按压回弹 |

### 1.3 四大视觉支柱(来源:`DESIGN.md` §Brand)
1. **Jelly-Texture(果冻质感)**:半透明虹彩渐变,模拟吉祥物的发光体质。
2. **Scrapbook Utility(手账实用)**:档案编号、印章、胶带,让情绪条目有「有序的混乱感」。
3. **Wink & Wit(俏皮机锋)**:柔和视觉 + 锋利微文案的反差。
4. **Ink Border(墨线描边)**:1px 深灰紫描边(`#4D4D4D`),让柔色保持沉稳可读,而非轻飘。

| 等级 | 建议色 | 理由 |

### 2.2 字体

#### 2.2.1 字体族(来源:`DESIGN.md` §Typography)

| 角色 | 字体 | 用途 |
|---|---|---|
| 标题 | **Plus Jakarta Sans** | 主标题/区块标题(软曲线、现代几何) |
| 正文 | **Be Vietnam Pro** | 长文、档案描述(高可读) |
| 标签/系统数据 | **Space Grotesk** | 档案编号 `#MM-xxx`、日期、毒舌旁白 |

#### 2.2.2 字号阶梯(来源:`DESIGN.md` frontmatter)

| Token | 字体 | 字号 | 字重 | 行高 |
|---|---|---|---|---|
| `headline-lg` | Plus Jakarta Sans | 32px | 700 | 40px(字距 −0.02em) |
| `headline-lg-mobile` | Plus Jakarta Sans | 28px | 700 | 36px |
| `headline-md` | Plus Jakarta Sans | 24px | 700 | 32px |
| `body-lg` | Be Vietnam Pro | 18px | 400 | 28px |
| `body-md` | Be Vietnam Pro | 16px | 400 | 24px |
| `label-sm` | Space Grotesk | 12px | 500 | 16px(字距 0.05em) |

> 小程序实现换算:1px ≈ 2rpx(750rpx 屏宽基准)。如 `headline-lg` 32px → 64rpx。

#### 2.2.3 🧪 建议:中文字体 fallback(小程序关键)

`DESIGN.md` 选用的是英文字体,而产品内容以中文为主。**小程序网络字体加载慢且体积受限**,建议:

```css
/* 标题 */
font-family: "Plus Jakarta Sans", "PingFang SC", "HarmonyOS Sans SC",
             "Source Han Sans CN", system-ui, sans-serif;
/* 正文 */
font-family: "Be Vietnam Pro", "PingFang SC", "HarmonyOS Sans SC",
             "Source Han Sans CN", system-ui, sans-serif;
/* 标签/编号(几何感,中文回落到无衬线即可) */
font-family: "Space Grotesk", "PingFang SC", "DIN Alternate",
             "HarmonyOS Sans SC", monospace, system-ui;
```

- **中文优先用系统字体**(iOS 苹方 / 安卓 思源/鸿蒙),保证性能与零加载延迟。
- 仅在「档案编号 / 英文标题」等少量场景考虑自定义字体,且需做子集化 + 本地预置。
- 严禁因等待网络字体出现「字体闪烁(FOIT/FOUT)」影响首屏。

### 2.3 间距(来源:`DESIGN.md` frontmatter `spacing`)

| Token | 值 | 用途 |
|---|---|---|
| `margin-page` | 24px(48rpx) | 页面左右安全边距(供贴纸/胶带出血) |
| `gutter-card` | 16px(32rpx) | 卡片间距 |
| `padding-inner` | 20px(40rpx) | 卡片内边距 |
| `stack-sm` | 8px(16rpx) | 紧凑堆叠 |
| `stack-md` | 16px(32rpx) | 常规堆叠 |
| `stack-lg` | 32px(64rpx) | 区块堆叠 |

### 2.4 圆角与形状(来源:`DESIGN.md` §Shapes + frontmatter `rounded`)

- **卡片:** 最小 **32px** 圆角(squircle 超椭圆,「软糯可捏」感)
- **按钮 / 输入框:** pill 全圆角(`9999px`)
- **胶带(Washi Tape):** 矩形 + 两端撕裂/锯齿(随机化,增加手账颗粒感)
- **贴纸:** 异形模切 + 细白偏移边 + 1px 墨线

tailwind 制圆角令牌(供参考):`sm .25rem` / `default .5rem` / `md .75rem` / `lg 1rem` / `xl 1.5rem` / `full 9999px`。

### 2.5 层级 Elevation(来源:`DESIGN.md` §Elevation,用色调堆叠 + 背景模糊,而非重投影)

| 级别 | 说明 | 参数 |
|---|---|---|
| L1 基底 | 奶油暖白背景 | `#FFFBF5` |
| L2 卡片 | 纯白 + 1px 墨线 + 极柔紫调投影 | 投影 ~10% 透明 / 20px 模糊 |
| L3 玻璃(交互/浮层) | 磨砂玻璃(模态、吉祥物气泡) | 背景模糊 15px / 不透明度 60% |
| L4 贴纸/胶带 | 扁平高对比、0 模糊投影,像「贴」在 UI 上 | — |

> 深度要「浅」,像桌上叠放的纸与薄玻璃。

## 3. 组件规范

> 组件名对齐 `CLAUDE.md` §4 目录结构:`MonsterCard` / `CaptureLoading` / `ExcusePanel` / `MicroTask` / `SharePoster` / `MonsterBadge`。每个组件给出:用法 / 样式 / 状态。

### 3.1 Buttons(按钮)
- **Primary 主按钮:** 紫→蓝渐变填充(`linear-gradient(135deg, #B197FC, #74C0FC)`),pill 全圆角,内发光。按压时「Q 弹回弹」缩放 `scale(0.95)`。
  - 用途:首页「开始收容」、结果页「生成分享卡」等核心动作。
- **Secondary 次按钮:** Ghost 幽灵样式,1px 墨线描边 + 小「印章」图标。
  - 用途:「再收容一只」「我完成了任务」等。

### 3.2 Cards(卡片)→ `MonsterCard`
- 纯白底 + 32px 圆角 + 1px 墨线 + 柔紫投影(L2)。
- **右上角必须有「档案编号」**,Space Grotesk 字体,格式 `#MM-902`(来源:`DESIGN.md` §Cards + `_9` 视觉)。
- 顶部中央可选和纸胶带图形,把卡片「贴」在屏上。
- 内容区承载怪兽档案字段(见 4.3)。

### 3.3 Chips(情绪标签)
- 果冻半透明填充,圆角 pill。
- **选中态:** 发光(glow)+ 标签旁出现爱心图标。
- 用途:首页快捷情绪(`我没时间` / `我好焦虑` / `我又拖延了` / `我不想干活`)。
  > 📌 截图 `_1` 显示 4 个标签;`prd` §9.1 提到 5 个(多一个 `我还没准备好`)。**以截图 4 个为准上线,第 5 个作为候选扩展**。

### 3.4 Input(输入框)
- 柔白填充 + 轻微内阴影,呈「凹陷」手写质感。
- 高行高,文案感觉「手写」。
- 右下角字数计数 `0/200`(限制单次输入长度,对齐 `CLAUDE.md` §8 成本控制)。
- placeholder:`写下此刻的心情,越具体越能被理解哦…`(来源:`_1` 视觉)。

### 3.5 The Mascot(吉祥物)→ `CaptureLoading` / 全局
- 居中或从边缘探出。
- 渐变:`linear-gradient(135deg, #B197FC 0%, #74C0FC 100%)`。
- **呼吸动画:** 持续轻微 `scale` 缩放 + 上下浮动。
- 用途:首页陪伴、加载页捕获动画主角、空状态。

### 3.6 Stickers & Stamps(贴纸与印章)→ `MonsterBadge`
- 装饰性状态指示:`CAPTURED`(已捕获)、`HEALED`(已治愈)、`TOXIC`(有毒)等。
- **5–10° 倾斜**,打破水平线,强化手账感。
- 扁平高对比、0 模糊投影(L4),像贴纸贴上。
- 用途:档案卡状态角标、任务完成印章。

### 3.7 ExcusePanel(借口粉碎面板)
- 仅 **借口类 / 拖延类** 怪兽显示(由 `monsterType` 决定)。
- 承载字段:`trueIdentity`(伪装鉴定)、`excuseCrush`(借口粉碎)、`doNotFeed`(今日禁止投喂)。
- 视觉:可用「证据贴纸」「划掉的借口」呈现拆穿感(毒舌 25% 的载体)。

### 3.8 MicroTask(5 分钟驯化任务)
- 承载 `microAction` 字段。
- 含「我完成了任务」按钮 → 完成后可生成「怪兽出院证明」分享卡。
- 视觉:步骤卡 / 打勾印章。

### 3.9 SharePoster(分享海报)
- 用 **Canvas** 生成图片(对齐 `CLAUDE.md` §2、§5)。
- 3 种类型(详见 4.4):今日怪兽卡 / 借口死亡证明 / 怪兽出院证明。
- 含产品 Logo / 水印,适配朋友圈/微信群竖图。

## 4. 页面设计

> 布局哲学(来源:`DESIGN.md` §Layout):**Mobile-First 竖屏 9:16 流式栅格**;左右 24px 安全边距(供贴纸/胶带出血);内容以浮动卡片组织;**顶部约 40% 为「怪兽互动区」,底部约 60% 为「档案/手账功能区」**;元素可适度重叠(贴纸压在卡角)打破数字栅格。

> 字段含义统一来自 `analyzeMood` 云函数返回结构(对齐 `CLAUDE.md` §3):`monsterName` / `monsterType` / `dangerLevel` / `catchphrase` / `trueIdentity` / `whyItAppears` / `whatItProtects` / `excuseCrush` / `doNotFeed` / `microAction` / `cardText{title,subtitle,line}`。

### 4.1 首页 index — 📐 推导 + `_1` 视觉
**职责:** 让用户秒懂产品并输入当前状态(`prd` §9.1)。

| 区域 | 内容 |
|---|---|
| 顶部 40% 怪兽互动区 | 主标题「心情怪兽收容所」+ 副标题「接纳每一种心情·陪伴每一个你」+ 吉祥物 + 气泡「来吧,我帮你收容今天的小怪兽~」 |
| 中部输入 | Input 输入框(placeholder「写下此刻的心情,越具体越能被理解哦…」,`0/200`) |
| 快捷 | 4 个 Chips(`我没时间`/`我好焦虑`/`我又拖延了`/`我不想干活`) |
| 主操作 | Primary 按钮「开始收容」→ 跳加载页 |
| 底部 | 5-tab 导航(见 §5)+ 安全声明页脚(见 §7) |

入口:小程序首页;出口:加载页 loading。

### 4.2 加载页 loading — 📐 推导(`prd` §9.2)
**职责:** 把模型生成等待转化为产品体验。

- 全屏吉祥物 + 捕获动画(网兜/怪兽影子/收容笼/扫描光效/档案印章)。
- **轮播文案**(随机):「正在巡逻心情走廊…」「发现一只可疑小怪兽…」「它正在伪装成『我没时间』…」「正在投喂镇定饼干…」「正在打开收容笼…」「收容成功!」
- 被动等待,**自动跳转**结果页(不要让用户手动点;等待用动画承接,对齐 `CLAUDE.md` §8)。

### 4.3 结果档案页 result — ✅ 基于 `_9` 成品 + 字段表
**职责:** 展示核心 Agent 输出(`prd` §9.3)。这是唯一有成品截图的页面。

| 区域 | 内容(字段映射) |
|---|---|
| 档案头 | 收容编号 `#MM-xxx`(Space Grotesk)+ 怪兽插画 + `monsterName` + `MonsterBadge`(`monsterType` 类型标签 + `dangerLevel` 危险等级,配色见 2.1.3) |
| 正文卡片(`MonsterCard`) | `catchphrase` 常见叫声 / `trueIdentity` 真实身份(伪装鉴定)/ `whyItAppears` 出现原因 / `whatItProtects` 它在保护你什么 |
| 借口区(`ExcusePanel`,仅借口/拖延类) | `excuseCrush` 借口粉碎 / `doNotFeed` 今日禁止投喂 |
| 行动区(`MicroTask`) | `microAction` 5 分钟任务 + 「我完成了任务」按钮 |
| 底部操作 | Primary「生成分享卡」/ Secondary「我完成了任务」「再收容一只」「加入图鉴」 |
| 装饰 | 顶部和纸胶带、`CAPTURED` 印章、贴纸(`_9` 视觉) |

入口:加载页;出口:分享页 / 图鉴 / 首页。

### 4.4 分享页 share — 📐 推导(`prd` §7.5、§9.4)
**职责:** 生成一张用户愿意截图/保存/转发的卡。3 类(对应 `SharePoster`):

| 卡类型 | 触发 | 内容 |
|---|---|---|
| 今日怪兽卡 | 情绪类结果 | `cardText.title`(默认「今日心情怪兽」)/ `subtitle`(怪兽名)/ 头像 / `catchphrase` / `trueIdentity` / 今日照顾 / 5 分钟任务 / Logo 水印 |
| 借口死亡证明 | 借口类结果 | 借口 / 死因(如「被『先做 v0』当场击毙」)/ 遗言 / 处理建议 |
| 怪兽出院证明 | 完成 5 分钟任务后 | 怪兽名 +「今日已稳定」/ 出院原因 / 复发提醒(如「明天可能伪装成『我再优化一下』」) |

> 📌 `prd` §19 把「借口死亡证明」排在 v0.2;v0.1 先做「今日怪兽卡」与「出院证明」。

入口:结果页;出口:系统分享 / 返回结果页。

### 4.5 图鉴页 gallery — 📐 推导(`prd` §9.5)
**职责:** 让产品从「一次性生成器」变成可持续记录的小工具。

- 已收容怪兽列表(头像 + 名 + 出现次数)+ 本周主导怪兽 + 高频借口 + 最近任务 + 已完成任务记录。
- **不做账号系统**,用 **localStorage**(Taro Storage)保存历史(对齐 `CLAUDE.md` §7)。

入口:结果页「加入图鉴」或底部 tab「记录」;出口:回看任务。

### 4.6 安全回应态 — 📐 推导(`prd` §15、`CLAUDE.md` §6)
**触发:** 用户输入涉及自伤、自杀、强烈绝望等高风险内容。

**UI 必须切换为与正常档案不同的态:**
- ❌ 不显示怪兽卡 / 借口粉碎 / 搞笑贴纸 / 分享卡。
- ✅ 显示:① 表达关心 ② 鼓励联系身边可信任的人 ③ 建议寻求专业帮助 ④ 提供紧急求助资源 ⑤ 安静、去娱乐化的视觉(降低吉祥物活泼度,改用柔和、克制的排版)。

> 视觉规范 `prd` 未细化;**建议**用 Creamy 底 + 柔和 Ink 文字 + 一只「安静陪伴」的吉祥物,无印章/无倾斜贴纸。

## 6. 动效规范

技术:CSS Animation + Taro 动画 API(对齐 `CLAUDE.md` §2)。原则:轻、软、Q 弹,不超过 300–400ms。

| 场景 | 动效 |
|---|---|
| 吉祥物 | 持续呼吸(`scale 1↔1.04`)+ 上下浮动 |
| 按钮按压 | `scale(0.95)` 回弹 |
| 加载页捕获 | 网兜落下 / 扫描光带 / 印章盖下(分阶段承接等待) |
| 卡片入场 | 上移 + 淡入(stagger 错落) |
| Chip 选中 | glow + 爱心弹入 |
| 印章盖戳 | 旋转入场 + 轻微回弹 |

> 性能:小程序避免大量并发动画;优先 `transform`/`opacity`;长列表避免逐项动画。

## 7. 文案与安全边界(强约束,对齐 `CLAUDE.md` §6)

### 7.1 语气 do / don't
- ✅ 说:解释背后的怪兽、给一个不痛苦的小行动(例:「这只怪兽不是坏,它只是想保护你不被否定」)。
- ❌ 禁说:「你就是懒」「你心理有问题」「你必须积极起来」「你需要自律」「你这是焦虑症」等。

### 7.2 安全声明页脚(常驻)
首页与结果页底部固定声明(来源:`prd` §15):
> 「心情怪兽收容所不是心理诊断工具,只是一个轻量自我观察和行动陪伴产品。如果你长期处于强烈痛苦、失眠、自伤念头或无法正常生活,请及时联系专业人士或身边可信任的人。」

### 7.3 高风险流程(5 步)
见 §4.6:停娱乐化 → 关心 → 鼓励联系 → 建议专业帮助 → 紧急求助资源;**不再生成搞笑怪兽卡**。

### 7.4 禁止
任何医疗/心理诊断(抑郁症、焦虑症、需服药等);不得包装成心理诊断或医疗工具(影响小程序审核)。

## 8. 落地 Taro 备注(对齐 `CLAUDE.md` §2、§3、§8)

| 项 | 要求 |
|---|---|
| 样式方案 | CSS Modules / Less;NutUI React 可作基础组件库 |
| 单位 | rpx(750rpx 屏宽基准);设计稿 px ÷ 2 ≈ rpx |
| 字体 | 中文优先系统字体(见 2.2.3),英文/编号可用自定义字体但需子集化 |
| 大模型调用 | **绝不在前端直连大模型**,一律走云函数 `analyzeMood`;API Key 只在云函数(`CLAUDE.md` §3) |
| 返回结构 | 云函数返回可直接渲染的结构化 JSON,前端不做复杂文本解析;JSON Schema 校验 + 兜底文案 + 重试一次(`CLAUDE.md` §3.4、§8) |
| 分享卡 | 小程序 Canvas 生成图片(`SharePoster`) |
| 本地存储 | Taro Storage / 微信小程序 Storage(图鉴历史,无登录) |
| 成本控制 | 限制输入长度(`0/200`)、控制输出长度、优先低成本模型 |
